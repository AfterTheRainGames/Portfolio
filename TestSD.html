  • Short Description: Small C# utility that reads version numbers from a local VersionChecker.docx and synchronizes local program folders with the matching latest release folders on a shared network location.

    Key Features:

    • Reads version info from a Word document table using Microsoft.Office.Interop.Word.
    • Finds latest shared-version folders for configured programs, compares versions and copies updated files to the local NETPrograms folder.
    • Updates the Word document with the new shared versions after copying and releases COM objects to avoid leaks.

    Technologies Used: .NET 8, C# 12, Microsoft.Office.Interop.Word, file I/O, UNC network paths, COM interop

    Outcome: Automates keeping local deployments in sync with the shared repository, reducing manual update work and deployment errors.

  • Short Description: Windows Forms application that lets users visually build automation scripts using Click, Type and Wait commands and run them; scripts are persisted as JSON files under Desktop/JSONs.

    Key Features:

    • Visual script builder: add Click, Type, and Wait steps with a simple UI.
    • Script management & execution: save/load/delete scripts as JSON, run asynchronously, simulate mouse/keyboard via P/Invoke.

    Technologies Used: C# (WinForms on .NET 10), System.Text.Json, P/Invoke to user32.dll, SendKeys, file I/O.

    Outcome: Users can create, persist, share, and run repeatable click/type/wait sequences to automate desktop tasks.
