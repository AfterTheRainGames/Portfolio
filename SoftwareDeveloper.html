Auto-Installer

Short Description: Small C# utility that reads version numbers from a local VersionChecker.docx and synchronizes local program folders with the matching latest release folders on a shared network location.

Key Features:

  • Reads version info from a Word document table using Microsoft.Office.Interop.Word.
  • Finds latest shared-version folders for configured programs, compares versions and copies updated files to the local NETPrograms folder.
  • Updates the Word document with the new shared versions after copying and releases COM objects to avoid leaks.

Technologies Used: .NET 8, C# 12, Microsoft.Office.Interop.Word, file I/O, UNC network paths, COM interop

Outcome: Automates keeping local deployments (e.g., EmailCounterPull and EmailCounterPush) in sync with the shared repository, reducing manual update work and deployment errors.

AutoClicker

Short Description: Windows Forms application that lets users visually build automation scripts using Click, Type and Wait commands and run them; scripts are persisted as JSON files under Desktop/JSONs.

Key Features:

  • Visual script builder: add Click (screen coordinates), Type (text or modifier keys), and Wait (seconds) steps with a simple UI and real-time cursor coordinate capture.
  • Script management & execution: save/load/delete scripts as JSON, dynamically list available scripts, run scripts asynchronously with optional looping and cancellation (End key), and simulate mouse/keyboard via P/Invoke and SendKeys.

Technologies Used: C# (WinForms on .NET 10), System.Text.Json for serialization, P/Invoke to user32.dll (GetCursorPos, mouse_event), SendKeys, and file I/O for JSON persistence.

Outcome: Users can create, persist, share, and run repeatable click/type/wait sequences to automate repetitive desktop tasks; scripts are human-readable and portable JSON files.

Email Counter

Short Description: A small automation tool that reads Outlook mail (inbox, subfolders, Sent and Deleted Items), counts sent and received emails per workday, and records the results into a personal Excel workbook — then pushes those per-day counts into a shared engineering Excel workbook so team-wide tracking is centralized.

Key Features:

  • Scans Outlook folders to compute daily sent/received counts and (where applicable) uses a membership database to identify team recipients/senders when counting deleted items.
  • Maintains a personal Excel file with per-day counts and also pushes those counts into a central/shared engineering Excel workbook.
  • Maps the current Excel/Windows user to the correct target column in the shared workbook (name→column mapping) and writes both Received and Sent columns for that user.
  • Ensures the shared table has at least as many rows as the personal table, adding rows if needed, and writes the date column with a consistent date format.
  • Opens the personal workbook read-only and writes only to the shared engineering workbook; aborts safely if table sizes are inconsistent to avoid corruption.
  • Properly releases COM objects and attempts to kill lingering Excel processes (uses Windows API to find Excel PID) to avoid orphaned instances.

Technologies Used: C# (.NET 8), Microsoft.Office.Interop.Outlook (reading), Microsoft.Office.Interop.Excel (read/push), Windows API (DllImport/GetWindowThreadProcessId), and standard .NET libraries for file, path and date handling.

Outcome: Automated daily email activity reporting that both produces local per-day sent/received counts and pushes those counts into a central Email_Counter_Data.xlsx for team-wide tracking — eliminating manual copy/paste and ensuring proper Excel cleanup after execution.
